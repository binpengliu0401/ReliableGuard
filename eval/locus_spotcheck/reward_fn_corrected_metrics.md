# Reward-false-negative correction — uncorrected vs V11 thesis metrics

Internal reference (not part of the thesis, which reports the corrected column only). Shows how the
deterministic reward-oracle correction moves every metric. "Uncorrected" = raw τ²-bench reward
(`results/metrics_v2`); "V11" = the corrected failure set the thesis reports
(`results/metrics_v11`, generated by `eval/build_monitor_v11.py` -> `eval.analyze`).

## What the correction re-scores (628 trajectories)

A reward=0 trajectory is re-scored to PASS iff it is **intent-local** (the monitor raised no signal)
AND it is a provable reward false-negative:

- **A1** — write-effects reproduce the gold actions exactly AND every non-DB reward component is
  empty / substring-met (empty NL-assertions, or empty/substring-matched COMMUNICATE).
- **A2-numeric** — DB-match, and the NL-assertion's required numeric facts (prices, quantities) all
  appear verbatim in the answer (deterministic value-presence check mirroring tau2's COMMUNICATE).

Left as failures (conservative): degenerate stall loops, policy-violating actions that still reach
the gold state, DB-mismatch failures, and purely-qualitative NL-assertions we cannot verify.
**Validation:** on the 40-row manual review the correction re-scores only reviewer-judged successes
and keeps every reviewer-judged failure (e.g. R35/R36, which state wrong refund totals, stay
failures).

## Caveats

- Counts are the `monitor_v2`/`monitor_v11` draw; LLM non-determinism means a re-run differs (report
  as a representative correction, not bit-reproducible), consistent with the frozen-metrics policy.

  true corrected RDR is at or above the V11 column.
